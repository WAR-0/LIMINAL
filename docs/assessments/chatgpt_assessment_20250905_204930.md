## Executive Summary
LIMINAL is best positioned as a physics-based memory architecture with consciousness research as a secondary, exploratory track. The core scientific value lies in a falsifiable engineering program: (1) build a stable screened-Poisson field substrate; (2) validate a semantics-preserving attention→mass interface; (3) demonstrate functional gains over strong baselines via a rigorous benchmarking suite. Consciousness claims should remain descriptive and clearly non-evidentiary.

Proceed with caution under the documented phase gates. The most material risks are: (a) the “Mass Salad” interface failure; (b) closed-loop instability; and (c) single-developer scope. These are mitigable through the published thresholds, ablation controls, and strict go/no-go discipline.

## Updated Assessment

### 1) Scientific Merit
- Functional theory: A continuous field can serve as a unified substrate for context persistence, attention bias, and identity stability. This is testable and worthwhile.
- Consciousness claims: Phenomenal sufficiency is not addressed. Φ-like metrics measure dynamical/integration complexity, not experience. Use them as secondary internal metrics only.
- Temporal binding: LIMINAL plausibly addresses functional binding (what to keep, when to recall) via rolling-field dynamics; it does not resolve phenomenal binding.

### 2) Technical Viability
- Field substrate: 256×256 @ ~10 Hz is feasible for MVP; FFT-based screened Poisson with CIC deposition and damping is standard numerics. If gains plateau, consider 512×512 or multiscale/adaptive approaches.
- Interface (make-or-break): Adopt the Phase 1 protocol exactly—require semantic–spatial correlation global >0.4 and local >0.6 with ablations (randomized projection, scrambled attention, frozen field) before closing the loop.
- Feedback discipline: Introduce influence only through bounded, interpretable channels (retrieval gating, attention reweighting). Add hysteresis, damping, and watchdogs; keep emergency fallbacks active.
- Identity stability: Test with longitudinal protocols (10+ hour sessions), blinded scoring, and degradation curves; compare to vanilla/buffer/RAG.

### 3) Research Value
- High if benchmarks show significant functional gains (memory, attention, consistency) under blinded evaluation. The benchmarking protocol itself is a strong contribution.
- Even negative results are valuable (e.g., interface fails, no gains over baselines), clarifying limits of physics-mediated architecture for LLM memory.

### 4) Risk Assessment
- Theoretical: Unfalsifiable consciousness; risk of anthropomorphic misreadings. Mitigate with blinded evaluation, null-hypothesis controls, and effect-size reporting.
- Practical: Interface brittleness, closed-loop instability, scope. Mitigate with phase gates, bounded coupling, rigorous monitoring, and modular delivery.
- Lineage risk: Treat DRIFT/ECHO/UNCAN as specs; do not assume inherited performance. Build empirical evidence within LIMINAL.

## Concrete Go/No-Go Gates (Reaffirmed)
- Gate A (Interface Viability): Semantic–spatial correlation thresholds met across multiple domains; ablations eliminate effect. If <0.3, no-go; if 0.3–0.4, remediate and retry once; if ≥0.4 global and ≥0.6 local, proceed.
- Gate B (Closed-Loop Stability): Perplexity ratio ≤1.5 vs baseline; attention variance below instability threshold with coupling ≤ capped value; recovery protocols verified.
- Gate C (Functional Gains):
  - Memory/Consistency: ≥15% over vanilla; within 5% of RAG on targeted tasks.
  - Attention/Focus: ≥25% drift reduction; topic relevance ≥90%.
  - Integration: ≥40% improvement over concatenation baseline; false connections <10%.
  - Efficiency: Overhead <50% vs vanilla; no leaks over 24h.
- Gate D (Stretch Targets): Toward 10–20× consolidation compression with ≥70–80% recall post 5k+ token distraction.

Failing any hard gate twice without clear remediation path warrants a pivot to “physics-assisted memory analytics/visualization” without closed-loop control.

- Maintain blinded human evaluations and randomized controls. Publish negative results.

## Prioritized Implementation Plan (Single-Developer Reality)
1) Engine MVP (2–3 weeks): Implement screened-Poisson solver (FFT), CIC deposition, damping, stability checks, and lightweight visualization. Defer advanced features.
2) Interface Validation (2–3 weeks): Build the attention→projection→mass path; instrument metrics; run domain-diverse, ablated studies. Do not enable feedback yet.
3) Controlled Feedback (2–3 weeks): Add retrieval gating and attention reweighting with tight coupling caps and hysteresis; verify stability and run initial benchmarks.
4) Benchmarking Pass (2–4 weeks): Execute the LIMINAL benchmark suite vs vanilla, buffer, RAG, static/random field controls; produce blinded, statistically powered reports.
5) Iterate or Pivot: If gates are passed with significant gains, scale resolution/parameters incrementally; if not, pivot to analytics/visualization and publish findings.

Timeboxing and strict scope control are essential. Avoid building noncritical features before Gate C results.

## Bottom Line
- Scientific Merit: Strong as a functional cognitive architecture investigation; not evidential for phenomenal consciousness.
- Technical Viability: Feasible MVP with disciplined coupling and robust controls; outcome hinges on the interface.
- Research Value: High if benchmarks confirm gains; otherwise, valuable negative evidence plus a reusable benchmarking/risk framework.
- Recommendation: Proceed under strict phase gates and evidentiary standards; keep consciousness claims descriptive and secondary; treat lineage projects as plans, not proof.
